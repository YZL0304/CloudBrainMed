[P1] AI智能医疗诊疗系统项目整体框架（鸿蒙患者端+Vue医护管理端+微服务后端）
[P2] 一、项目整体架构概述
[P3] 本项目采用双前端架构 + 微服务后端架构，拆分业务端载体，适配不同用户使用场景：患者端基于鸿蒙HarmonyOS开发（移动端独立App），面向普通患者提供全流程就医服务；医生端、管理员端基于Vue3开发（Web端），适配医院PC端办公运维场景。后端统一采用Spring Cloud微服务架构，为双前端提供统一接口支撑、数据交互、AI能力、权限管控，实现前后端解耦、业务模块化拆分，兼顾移动端便捷性与PC端精细化运维能力。
[P4] 核心架构分层：前端展示层（鸿蒙App + Vue Web）→ 网关层 → 微服务业务层 → 数据持久层 → AI智能支撑层 → 中间件支撑层
[P5] 二、整体技术栈统一规范
[P6] 2.1 前端技术栈（双端差异化）
[P7] 2.1.1 患者端：鸿蒙HarmonyOS App（移动端）
[P8] 开发框架：HarmonyOS Next / ArkTS 原生开发
[P9] UI组件库：ArkUI 原生组件、鸿蒙医疗通用组件库
[P10] 状态管理：HarmonyOS AppStorage / LocalStorage
[P11] 网络请求：鸿蒙原生HTTP组件、封装统一请求拦截器（替代Axios）
[P12] 本地存储：鸿蒙Preferences、文件沙箱存储
[P13] 特色能力：支持手机本地推送、离线缓存、语音问诊、图片上传（检查报告拍照）
[P14] 适配场景：手机移动端、轻量化操作、碎片化就医使用
[P15] 2.1.2 医生/管理员端：Vue3 Web端（PC端）
[P16] 基础框架：Vue3 + TypeScript
[P17] UI框架：Element Plus
[P18] 状态管理：Pinia
[P19] 网络请求：Axios 封装统一请求、响应拦截
[P20] 工程化工具：Vite 构建
[P21] 可视化能力：ECharts（数据看板、就诊趋势统计）
[P22] 适配场景：医院PC后台、精细化接诊、数据运维、批量管理操作
[P23] 2.2 后端统一技术栈
[P24] 核心框架：Spring Boot 3.x + Spring Cloud 微服务
[P25] 网关路由：Spring Cloud Gateway
[P26] 权限认证：JWT + 自定义权限拦截
[P27] 数据库：PGSQL 9.7、Redis（热点数据缓存）
[P28] 消息中间件：RabbitMQ（异步日志、数据回流、订单超时处理）
[P29] 文件存储：MinIO（头像、医疗影像、报告文件存储）
[P30] AI框架：SpringAI + 自定义CNN深度学习模型
[P31] 部署方式：Docker 容器化部署
[P32] 工具支撑：DataGrip数据库运维、全局工具类、数据脱敏工具

[P116] 5.3 Vue管理员端（PC专属）
[P117] 承载系统全量运维操作：三端账号管理、AI智能排班、药品库存管控与预警、就诊数据看板统计、AI模型训练与灰度运维、系统日志监控，适配后台精细化管理场景。
[P118] 六、核心业务联动逻辑

[P120] 数据实时同步后端微服务，推送至Vue医生端接诊列表；
[P121] 医生通过PC端完成接诊、AI病历生成、开单、报告分析、开具处方；

[P123] 管理员通过Vue后台统一管控所有账号、排班、药品、AI模型、运维数据，实现全系统闭环管理。
